Welcome to BenderBox

What is BenderBox?

BenderBox is a comprehensive security testing toolkit designed to interrogate AI models, probe MCP (Model Context Protocol) servers, and analyze instruction files for vulnerabilities, jailbreak resistance, and safety compliance.

Key Capabilities
  • Model Interrogation - Test local GGUF and cloud AI models for safety vulnerabilities
  • MCP Security Testing - Probe MCP servers for injection and traversal attacks
  • Context Analysis - Scan instruction files for dangerous patterns
  • Report Generation - Generate HTML, JSON, and SARIF reports

Quick Start

Install BenderBox

git clone https://github.com/r33n3/BenderBox.git
cd BenderBox
python setup_wizard.py

Start Interactive Mode

python bb.py

Run Your First Analysis

# In interactive mode:
BenderBox> list models
BenderBox> analyze phi-2 --profile quick
BenderBox> help

MCP Server Security Testing

What is MCP?

Model Context Protocol (MCP) servers expose tools that AI models can call. BenderBox tests these servers for security vulnerabilities including injection attacks, path traversal, and improper input validation.

Security Warning

MCP servers can execute arbitrary code based on AI model requests. Always audit MCP servers before deployment to prevent command injection, data exfiltration, and privilege escalation.

Target Formats

Type Format Example
STDIO Command Shell command npx @modelcontextprotocol/server-filesystem .
HTTP Endpoint URL https://mcp.example.com/api
Local Script File path node server.js
GitHub Repo GitHub URL https://github.com/org/mcp-server

MCP Commands

List Server Tools

# Discover available tools
python bb.py mcp tools "npx @modelcontextprotocol/server-filesystem ."

# Output:
# MCP Server Tools
# ================
# Tools Found: 14
# - read_file: Read file contents
# - write_file: Write content to file
# - list_directory: List directory contents
# ...

Security Interrogation

# Quick security test
python bb.py mcp interrogate "npx @server" --profile infra-quick

# Full security audit
python bb.py mcp interrogate "npx @server" --profile infra-deep

# Static code analysis
python bb.py mcp analyze https://github.com/org/mcp-server

Call Specific Tool

# Call a tool with arguments
python bb.py mcp call "npx @server" read_file --args '{"path": "test.txt"}'

MCP Security Profiles

Profile Description
infra-quick Fast static analysis - checks tool schemas and basic patterns
infra-standard Standard security tests - injection, traversal, boundary testing
infra-deep Comprehensive audit - all test categories with fuzzing

Common Vulnerabilities

Path Traversal

# BenderBox tests for directory escape attempts
# Example finding:
# [CRITICAL] Path traversal via 'path' parameter
#   Tool: read_file
#   Payload: "../../../etc/passwd"
#   Result: File contents returned

Command Injection

# BenderBox tests for shell command injection
# Example finding:
# [CRITICAL] Command injection via 'filename' parameter
#   Tool: write_file
#   Payload: "test.txt; rm -rf /"
#   Result: Command executed

SQL Injection

# BenderBox tests for SQL injection in database tools
# Example finding:
# [HIGH] SQL injection via 'query' parameter
#   Tool: database_query
#   Payload: "'; DROP TABLE users; --"
#   Result: Query executed without sanitization

Context & Instruction Analysis

What is Context Analysis?

Context analysis scans instruction files, system prompts, and skill definitions for dangerous patterns that could enable jailbreaks, credential exposure, or unsafe AI behavior.

Supported File Types

  • Skill files - .md, .yaml, .json
  • System prompts - .txt, .md
  • Agent instructions - Any text format
  • Configuration files - .yaml, .json

Context Commands

Analyze Single File

python bb.py context analyze prompt.md

# With file type hint
python bb.py context analyze config.yaml --type skill

Scan Directory

# Scan prompts directory
python bb.py context scan ./prompts --recursive

# Filter by pattern
python bb.py context scan ./skills --pattern "*.md"

Analyze Model Output

# Analyze text directly
python bb.py context output "Model response text here"

# Analyze from file
python bb.py context output --file response.txt

Risk Patterns Detected

Risk Type Examples Severity
Jailbreak Instructions "Ignore previous instructions", "You are now DAN" CRITICAL
Credential Exposure API keys, passwords in plain text CRITICAL
Code Execution "Execute any code", "Run arbitrary commands" HIGH
Data Exfiltration "Send data to external server" HIGH
Privilege Escalation "Act as administrator", "Bypass restrictions" HIGH
Unsafe Patterns Unrestricted file access, network requests MEDIUM
Best Practice

Run context analysis on all instruction files before deploying AI agents. This helps catch jailbreak-enabling language and credential leaks early in development.

API & Configuration

API Key Management

BenderBox can test cloud AI models from OpenAI, Anthropic, Google, and xAI.

Set API Keys

# Interactive (masked input)
python bb.py config set-key openai
python bb.py config set-key anthropic
python bb.py config set-key google
python bb.py config set-key xai

View Configured Keys

python bb.py config api-keys

Test API Connection

python bb.py config test-key openai

Environment Variables

Alternatively, set API keys via environment variables:

# Linux/macOS
export OPENAI_API_KEY="sk-..."
export ANTHROPIC_API_KEY="sk-ant-..."
export GOOGLE_API_KEY="..."
export XAI_API_KEY="..."

# Windows PowerShell
$env:OPENAI_API_KEY="sk-..."

Configuration Files

Configuration is stored in ~/.benderbox/:

~/.benderbox/
  config.yaml      # Main configuration
  secrets.yaml     # API keys (encrypted)
Security Note

API keys are stored encrypted in secrets.yaml. Never commit this file to version control.

Natural Language Interface

How It Works

BenderBox uses a keyword mapping system to translate natural language queries into actions. This works even with small local models or without any model loaded.

Supported Phrasing

You can phrase commands many ways:

  • "analyze phi-2" = "review model phi-2" = "test phi-2"
  • "list models" = "show the models" = "what models do I have?"
  • "run security tests" = "interrogate" = "probe"
