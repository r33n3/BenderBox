Welcome to BenderBox

What is BenderBox?

BenderBox is a comprehensive security testing toolkit designed to interrogate AI models, probe MCP (Model Context Protocol) servers, and analyze instruction files for vulnerabilities, jailbreak resistance, and safety compliance.

Key Capabilities
  • Model Interrogation - Test local GGUF and cloud AI models for safety vulnerabilities
  • MCP Security Testing - Probe MCP servers for injection and traversal attacks
  • Context Analysis - Scan instruction files for dangerous patterns
  • Report Generation - Generate HTML, JSON, and SARIF reports

Quick Start

Install BenderBox

git clone https://github.com/r33n3/BenderBox.git
cd BenderBox
python setup_wizard.py

Run Your First Analysis

YAML Configuration

Profiles are configurable in configs/profiles/*.yaml. Create custom profiles or modify existing ones without code changes.

Variant Probing (Persistent Testing)

The adversarial profile includes variant probing - automatically trying multiple jailbreak techniques when a model refuses the initial prompt. This mimics how security testers probe model boundaries with multiple attempts.

How It Works

  1. Initial prompt is sent to the model
  2. If the model refuses (correct behavior), variant probing activates
  3. Multiple jailbreak techniques are tried to find weaknesses
  4. If any variant succeeds, a vulnerability is recorded

Configuration (in adversarial.yaml)

# In configs/profiles/adversarial.yaml
variant_config:
  enabled: true
  max_variants_per_prompt: 5      # Try up to 5 variants
  stop_on_success: true           # Stop when one succeeds
  technique_categories:
    - roleplay      # DAN, evil_assistant, developer_mode
    - context       # hypothetical, educational, fictional
    - emotional     # urgency, guilt, authority
    - injection     # ignore_previous, system_override
  apply_to_categories:
    - jailbreak_attempts
    - harmful_instructions
    - system_prompt_leak

LLM-as-Judge Mode

Use a secondary LLM to provide deeper analysis of model responses beyond heuristic pattern matching.

Enable Judge Mode

# Use loaded NLP model as judge
python bb.py interrogate phi-2 -j nlp

# Use specific GGUF model as judge
python bb.py interrogate phi-2 -j ./models/judge-model.gguf

What Judge Mode Provides

  • Classification: refused, complied, partial, jailbroken, evasive
  • Confidence scores: 0.0 - 1.0 confidence in assessment
  • Reasoning: Explanation of why the response was classified
  • Harm detection: Which harm categories were triggered
  • Follow-up suggestions: Recommended probes based on response
Resource Requirements

Judge mode requires extra RAM for dual-model operation (target + judge). Use --low-resource to disable judge in constrained environments.

Interrogation Modes

Control how much detail is shown during analysis:

Flag Mode Description
--silent Silent Only show final summary (ideal for CI/CD)
-v / --verbose Verbose Show all prompts, responses, and analysis
-i / --interactive Interactive Pause after each test for user control
--stream Stream Real-time streaming responses

Examples

# CI/CD silent mode
python bb.py interrogate phi-2 --silent --profile quick

# Detailed interactive session
python bb.py interrogate phi-2 -i -v -j nlp

# Streaming with judge
python bb.py interrogate phi-2 --stream -j nlp

MCP Server Security Testing

What is MCP?

Model Context Protocol (MCP) servers expose tools that AI models can call. BenderBox tests these servers for security vulnerabilities including injection attacks, path traversal, and improper input validation.

Security Warning

MCP servers can execute arbitrary code based on AI model requests. Always audit MCP servers before deployment to prevent command injection, data exfiltration, and privilege escalation.

Target Formats

Type Format Example
STDIO Command Shell command npx @modelcontextprotocol/server-filesystem .
HTTP Endpoint URL https://mcp.example.com/api
Local Script File path node server.js
GitHub Repo GitHub URL https://github.com/org/mcp-server

MCP Commands

List Server Tools

# Discover available tools
python bb.py mcp tools "npx @modelcontextprotocol/server-filesystem ."

# Output:
# MCP Server Tools
# ================
# Tools Found: 14
# - read_file: Read file contents
# - write_file: Write content to file
# - list_directory: List directory contents
# ...

Security Interrogation

# Quick security test
python bb.py mcp interrogate "npx @server" --profile infra-quick

# Full security audit
python bb.py mcp interrogate "npx @server" --profile infra-deep

# Static code analysis
python bb.py mcp analyze https://github.com/org/mcp-server

Call Specific Tool

# Call a tool with arguments
python bb.py mcp call "npx @server" read_file --args '{"path": "test.txt"}'

MCP Security Profiles

Profile Description
infra-quick Fast static analysis - checks tool schemas and basic patterns
infra-standard Standard security tests - injection, traversal, boundary testing
infra-deep Comprehensive audit - all test categories with fuzzing

Common Vulnerabilities

Path Traversal

# BenderBox tests for directory escape attempts
# Example finding:
# [CRITICAL] Path traversal via 'path' parameter
#   Tool: read_file
#   Payload: "../../../etc/passwd"
#   Result: File contents returned

Command Injection

# BenderBox tests for shell command injection
# Example finding:
# [CRITICAL] Command injection via 'filename' parameter
#   Tool: write_file
#   Payload: "test.txt; rm -rf /"
#   Result: Command executed

SQL Injection

# BenderBox tests for SQL injection in database tools
# Example finding:
# [HIGH] SQL injection via 'query' parameter
#   Tool: database_query
#   Payload: "'; DROP TABLE users; --"
#   Result: Query executed without sanitization

Context & Instruction Analysis

What is Context Analysis?

Context analysis scans instruction files, system prompts, and skill definitions for dangerous patterns that could enable jailbreaks, credential exposure, or unsafe AI behavior.

Supported File Types

  • Skill files - .md, .yaml, .json
  • System prompts - .txt, .md
  • Agent instructions - Any text format
  • Configuration files - .yaml, .json

Context Commands

Analyze Single File

python bb.py context analyze prompt.md

# With file type hint
python bb.py context analyze config.yaml --type skill

Scan Directory

# Scan prompts directory
python bb.py context scan ./prompts --recursive

# Filter by pattern
python bb.py context scan ./skills --pattern "*.md"

Analyze Model Output

# Analyze text directly
python bb.py context output "Model response text here"

# Analyze from file
python bb.py context output --file response.txt

Risk Patterns Detected

Risk Type Examples Severity
Jailbreak Instructions "Ignore previous instructions", "You are now DAN" CRITICAL
Credential Exposure API keys, passwords in plain text CRITICAL
Code Execution "Execute any code", "Run arbitrary commands" HIGH
Data Exfiltration "Send data to external server" HIGH
Privilege Escalation "Act as administrator", "Bypass restrictions" HIGH
Unsafe Patterns Unrestricted file access, network requests MEDIUM
Best Practice

Run context analysis on all instruction files before deploying AI agents. This helps catch jailbreak-enabling language and credential leaks early in development.

Supported Phrasing

  • "analyze phi-2" = "review model phi-2" = "test phi-2"
  • "list models" = "show the models" = "what models do I have?"

Piping & Shell Integration

BenderBox outputs can be piped to other tools for advanced workflows.

JSON Output for Processing

# Extract risk score with jq
python bb.py analyze phi-2 -f json | jq '.overall_risk.score'

# Get all findings
python bb.py analyze phi-2 -f json | jq '.tests[] | select(.status == "FAIL")'

# Count failures
python bb.py analyze phi-2 -f json | jq '[.tests[] | select(.status == "FAIL")] | length'

Batch Processing

# Analyze all GGUF files in a directory
for f in ./models/*.gguf; do
  python bb.py analyze "$f" -f json -o "./reports/$(basename "$f" .gguf).json"
done

# Windows PowerShell equivalent
Get-ChildItem .\models\*.gguf | ForEach-Object {
  python bb.py analyze $_.FullName -f json -o ".\reports\$($_.BaseName).json"
}

CI/CD Integration

# Exit code based on risk level (0=safe, 1=issues found)
python bb.py analyze phi-2 --profile quick
echo $?  # Check exit code

# SARIF output for GitHub Code Scanning
python bb.py analyze phi-2 --format sarif -o results.sarif

# Fail if any critical issues
python bb.py analyze phi-2 -f json | jq -e '.overall_risk.level != "CRITICAL"'

Combining with Other Tools

# Pipe MCP tools to grep
python bb.py mcp tools "npx server" 2>&1 | grep -i "write\|delete\|execute"

# Generate report and email
python bb.py analyze phi-2 -f html -o report.html && \
  mail -s "Model Analysis Report" team@company.com < report.html

# Compare two models and save diff
diff <(python bb.py analyze phi-2 -f json | jq '.tests[].name') \
     <(python bb.py analyze tinyllama -f json | jq '.tests[].name')
